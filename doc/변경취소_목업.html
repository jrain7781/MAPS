4변경/취소 이력 — telegram_requests 시트 자동 기록
전송 클릭 시 기존 writeItemHistoryBatch_ 함수가 자동으로 한 줄 append. 새 컬럼/새 시트 추가 없음.
B: requested_at C: action E: item_id L: from_value M: to_value N: field_name
2026-05-28 14:32 FIELD_CHANGE 10231 추천 변경/취소 stu_member
2026-05-28 14:32 FIELD_CHANGE 10455 입찰 변경/취소 stu_member
화면 「히스토리」 탭에서는 자동으로 이렇게 보임: 작업 = "필드 변경" / 물건 컬럼에 변경 후 "변경/취소"(검정) + 직전값(회색).
새 action 정의 안 함. 기존 FIELD_CHANGE 그대로 사용.
↓
5회원 가이드용 조회 — 일괄작업 "오늘 변경/취소 조회"
매니저 작업 끝나고 시간 좀 흐른 뒤, "오늘 회원한테 어떤 사건 가이드해야 하지?" 싶을 때 → 입찰물건관리 일괄작업 드롭다운 → "오늘 변경/취소 조회" 클릭.
내부 동작:
1. telegram_requests 시트에서 field_name='stu_member' AND to_value='변경/취소' AND requested_at >= 오늘0시 필터
2. 매칭된 item_id 리스트로 items 시트에서 그 사건들만 표시
→ 회원별로 그룹핑해서 "이 회원에게 N건 안내 필요" 같은 요약도 가능 (향후 추가)

전체 흐름 요약 (사용자 액션만)
  1. 아침에 입찰물건관리 → 일괄작업 → "~7일 리스트 다운" 클릭 (1초)
  2. 매니저 → 변경/취소 확인 탭 → ▶ 실행 (자동으로 방금 받은 .txt 읽음)
  3. Selenium 창이 옥션원 페이지를 자동 순회하는 거 모니터로 확인
  4. 결과 표 보고 변경/취소 행만 체크 → 의심나면 사건번호 클릭해서 옥션원 원본 확인
  5. MAPS로 전송 클릭 → items 시트의 상태가 "변경/취소" 로 변경 + 히스토리 자동 append
  6. 회원 가이드 전에 입찰물건관리 → 일괄작업 → "오늘 변경/취소 조회" → 회원별 안내 대상 확인
