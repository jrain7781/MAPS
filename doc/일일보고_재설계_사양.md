# 일일보고 재설계 사양 (진행사항 확인 / 옥션원 크롤링 매니저)

작성 2026-06-02. 사용자 구두 사양 정리본. 구현 전 이 문서 기준으로 진행.

## 데이터 모델 (현재 존재 확인됨)
- 결과 row(`ccResults`/RESULT json): `sakun_no, court, bid_date, state_kind('불가'|'매각'|''), status,
  maegak_price(실제 매각/낙찰가), buyer, bidprice(우리 입찰가, 없을 수 있음), m_name(회원명),
  m_name_id(입찰담당자 이름 텍스트), detail, view_url, screenshot_path, is_buga, key_match …`
- `bidprice` "없을 수도" → 미입찰/낙찰 구분 불가 시 **"확인불가"** 처리.
- 매니저: `00.auction1/mj_tabs.js`(브라우저), `crawler.py`(서버), `report_builder.py`(PDF),
  `04.cc.py`(크롤러, 매 실행 새 프로세스), GAS `TelegramService.js`.

## 1. 분류 (최종)
물건 1건당:
- **불가**: `state_kind=='불가'` → 이미지 **검정**
- **매각**(`state_kind=='매각'`)일 때 입찰가 비교:
  - `bidprice` 없음 → **확인불가**
  - `maegak == bidprice` → **낙찰** → 이미지 **파랑**
  - `maegak <  bidprice` → **미입찰** (우리가 더 비싸게 쓸 거였는데 안 써서 놓침) → 이미지 **빨강**
  - `maegak >  bidprice` → 일반(우리보다 높게 팔림). 이미지 없음.
- **입찰(전체)** = 불러온 물건 전체 건수 (분모 개념, 카운트만)

## 2. 달력 + 날짜별 저장/조회
- 매칭 결과를 **날짜키별 저장**(현재 단일 localStorage `mj_cc_saved` → `mj_cc_saved_byDate[YYYY-MM-DD]`).
- 달력: 각 날짜에 매칭 건수 뱃지, 날짜 하단 낙찰/불가 건수.
- **날짜 클릭 → 그날 매칭 자료를 결과 리스트로 복원 표시**.

## 3. PDF = 일일보고 양식 전용
- 헤더: `YYYY년 MM월 DD일 일일보고` (기존 요약/배너 썸머리 제거).
- 집계 한 줄: **낙찰 N · 입찰 N · 불가 N · 미입찰 N**.
- 리스트: 각 건 (결과 = 매각/낙찰/불가) 한 줄씩.
- 리스트 뒤 이미지 카드: **낙찰 → 미입찰 → 불가 순**.
- 카드 헤더/테두리 색: 낙찰=파랑, 미입찰=빨강, 불가=검정.

## 4. 자동보고 스케줄러
- 시각 선택 UI(예: 14:00). 지정 시각에 자동:
  오늘 입찰건 불러오기 → 자동매칭 → 셋팅된 담당자에게 텔레그램 전송.
- 구현 위치 후보: 서버(crawler.py) 타이머 또는 매니저 상주. (구현 시 결정)

## 5. 자동보고 옆 "매칭 조사 히든/보이게" 옵션
- 크롤링 시 브라우저 표시 여부(04.cc.py headless 토글). 자동보고 토글 옆 배치.

## 6. 담당자(m_name_id)별 보고 분리
- 결과표/카드에 **담당자(m_name_id)** + 입찰가 표시.
- **m_name_id 별로 각각 따로** 보고서 생성·전송.

## 7. 수신 대상 모델 (확정)
- **관리자**(gubun=관리자, 예: 이정우): **전체 담당자 건 모두** 수신 (전체 일일보고).
- **강사/담당자 본인**: 자기 `m_name_id` 건만 수신.
- **체크박스 UI**: 담당자(강사) 목록에서 받을/안받을 선택 → 저장(재사용).
- m_name_id(이름 텍스트) → telegram chat_id 매핑은 회원명 기준 매칭.

## 8. UI 추가
- **보고자(수신 대상) 상단 상시 표시** (전송 모달 밖, 화면 상단 고정).
- 회원/담당자 이름 **앞에 텔레그램 뱃지**(토큰/사용 가능 여부).
- **불가 매칭 시 즉시 전송 버튼**: 해당 행에서 그 회원의 담당자에게 바로 텔레그램 전송.

## 9. 담당자 매칭 키 (중요)
- `m_name_id`는 담당자 **이름/닉네임 텍스트** (닉네임은 변경될 수 있음).
- 정확 매칭: 이름으로 `회원구분=강사` 회원을 서치 → 그 **회원 id**를 얻어 매칭 키로 사용.
- **매칭/그룹/전송 키는 항상 회원 id** (닉네임 변경에 안전). 화면 표시는 닉네임.
- 구조적 한계 인정: 원래 items에 강사 id가 들어가야 정상. 현재는 닉네임을 items에 매칭하는 구조 → 이름→회원 id 룩업으로 보완.
- 담당자는 **결과 리스트(표)에도** 컬럼으로 표시.

## 10. 이미지 복사하기
- 즉시 전송 버튼 옆에 **이미지 복사하기** 버튼: 해당 카드 이미지를 클립보드로 복사
  → 사용자가 카톡 등에 바로 붙여넣기 전송. (navigator.clipboard.write ClipboardItem PNG)

## 구현 단계(제안)
- P1: 분류 로직(확인불가/미입찰/낙찰/불가) + 날짜키 저장 구조.
- P2: 일일보고 PDF 재설계(report_builder) + 색상(검정/빨강/파랑).
- P3: 달력 UI + 날짜별 복원.
- P4: 담당자별 분리 전송 + 수신대상 체크박스/상시표시 + 텔레그램 뱃지.
- P5: 즉시 전송 버튼(불가).
- P6: 자동보고 스케줄러 + 히든/보이게 토글.
