## 1. 전체 그림 (불가/폐기 물건상태 모델)

- **상태값(stu_member)**: `폐기`(내부 불필요, 빨강) / `불가`(법원 진행불가, 검정). 기존 `취소`·`변경` 제거됨(사용자가 DB에서 전부 `폐기`로 통합 완료).
- **세부 2열 (items 시트 맨끝)**: `Z(26)=stu_reason`(불가사유: 변경/취소/취하/정지/연기/기각), `AA(27)=stu_reason_detail`(상세/종결문구). 기존 열 절대 손대지 말 것 — [[feedback_items_columns_append_only]]. (X=24 chuchen_read, Y=25 chuchen_read_date)
- **키 = (사건번호, 입찰일자, 법원) 3개.** item_id 안 씀(사용자 결정). 옥션은 사건번호로만 검색 → 가져와서 3키 일치 검증.
- **법원 변환**: 옥션 결과 주소 → `00.imageup/court_jurisdiction.py` 의 `find_court` → MAPS 법원명 (정변환). 역변환 안 함.

### #2 03.cc.py 옥션 실제 DOM 셀렉터 보정 (★ 로컬 테스트 필수)
1차 버전은 화면 추정으로 작성됨. 실제 옥션 로그인해서 돌려보고 보정:

- `parse_jongryo(detail_text)` — "본사건은 ○○(으)로 경매절차가 종결되었습니다" 정규식. 실제 문구 확인.
- `open_detail_text` — `auction/ca_view.php?product_id=` / `pubauct/view.php?product_id=` 가 맞는 상세 URL인지, body 텍스트에 주소/종결문구/기일 다 있는지.
- `date_hit` (입찰일자 일치) — 상세에 매각기일 표기 포맷 확인.
- 디버그: 실행 로그의 `[경매 i/N]`, `→ ✅/⚠키불일치` 라인 + 안 잡힌 필드 보고 받아 보정.
