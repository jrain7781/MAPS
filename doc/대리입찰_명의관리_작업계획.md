# 대리입찰 명의관리 + 출력 작업계획

작성일 2026-06-19 · 트리거 "대리입찰 명의관리 이어"

## 배경/목적
다물건 입찰 시 회원의 **본인/명의별 인적사항**(구분·이름·등급·전화·주민/법인번호·주소·계좌)을 관리하고,
이를 근거로 **위임장·기일입찰표(개인/법인)** 를 자동 출력한다.

## 확정 사양 (사용자 확인됨)
- **진행비**: 기존 자동계산 로직 그대로 — **건들지 않음**
- **참조 양식**: `00.imageup/다물건/대리입찰 샘플.xlsx` 의 **위임장-개인/법인, 기일입찰표-개인/법인 4종만** 참조 (입찰리스트·Sheet2 무시)
- **마스킹**: 카드 표시 시 주민번호/계좌 마스킹(뒷자리 가림), **저장은 원문, 출력(위임장·입찰표)은 원문**
- **직업** 컬럼 추가, 디폴트 `회사원` · **대리인-본인 관계** 상수 `지인`
- **저장 구조**: members 컬럼 증설 대신 **별도 시트 `member_accounts` 정규화**

## 데이터 설계 — `member_accounts` (신규 시트)
| 컬럼 | 의미 |
|---|---|
| member_id | 회원 FK |
| idx | 0=본인, 1~10=명의(nameN 매핑) |
| gubun | 개인/법인 (본인 디폴트 개인, 편집가능) |
| name | 본인=member_name / 명의=nameN |
| job | 직업 (디폴트 회사원) |
| jumin_corp | 개인=주민등록번호 / 법인=법인등록번호 |
| biz_no | 법인=사업자등록번호 (개인 공란) |
| phone | 전화번호 |
| address | 주소 |
| account_bank / account_no / account_name | 계좌 |
| update_date | 갱신일시 |

- 본인(idx0): name=member_name, phone/address/account_* 는 members 본인값으로 **시드**(없으면 빈값), 주민번호는 신규입력
- 명의(idxN): name=nameN, gubun=nameN_gubun, 나머지 신규입력
- **등급은 저장 안 함** → members.class_id → class 시트 `class_type - class_name - class_grade - class_loc` 연계

### 출력 매핑 (참고)
- 기일입찰표 *주민(사업자)등록번호*란 ← 개인: jumin_corp(주민번호) / 법인: biz_no(사업자번호)
- 기일입찰표 *법인등록번호*란 ← 법인: jumin_corp(법인등록번호)
- 대리인(수임자) 정보 ← **입찰대리인(직원)의 본인 레코드(idx0)** 에서 (성명/직업/주민번호/전화/주소/계좌)

## 단계별 진행 (단계마다 보고 → 사용자 테스트 → 다음)

### ✅ Phase 1 — 백엔드 데이터 기반  ← 이번
- `member_accounts` 시트 생성/헤더 보장
- `getMemberAccounts(memberId)`: 본인+명의 목록 반환(members 명의 + member_accounts 상세 머지, 본인 시드, 등급 연계)
- `saveMemberAccount(memberId, idx, data)`: (member_id, idx) upsert
- 자체검증 함수로 생성/저장/조회 로그 확인 후 보고

### Phase 2 — 우측 슬라이드 명의 카드 (조회/표시)
- 그리드에서 회원/명의 선택 → 우측에서 카드 슬라이드인
- 본인/명의 분기, 등급 연계, 마스킹 표시, 법인 시 사업자번호/법인등록번호 칸
- 읽기전용(표시) 우선 → 테스트

### Phase 3 — 카드 입력/저장
- 편집 + 저장 → saveMemberAccount, 개인/법인 분기 입력, 마스킹 표시·원문 저장
- 빈값 채우고 저장 → 재조회 유지 테스트

### Phase 4 — 위임장·기일입찰표 출력 (개인/법인)
- 카드 데이터 + 사건정보(법원/지원 파싱, 금액 자리분해)로 4종 양식 생성
- 출력 형식(인쇄용 HTML / xlsx)은 착수 시 확정 → 샘플 대조 테스트
