# 낙찰 카페글 생성 (어그로)

## 목적
다른 회원/비회원이 글을 보고 **"나도 경매하고 싶다 · 이렇게 돈을 버는구나 · 부럽다"** 를 느끼게 하는 후킹(어그로) 콘텐츠. 우리 회원의 낙찰 성공을 **증빙과 함께** 자랑스럽게, 그러나 **허위·과장 없이**.

## 1) 데이터 읽기 (크롤링 매니저)
Chrome MCP로 `http://localhost:8765` → 물건캡쳐 ▸ 낙찰카페등록 탭을 읽는다. 사용자가 [📥불러오기]+[▶실행]을 이미 눌러 데이터가 연동된 상태여야 한다(실행 안 됐으면 사용자에게 먼저 실행하라고 안내).
- 연동 데이터(폼 input value): `ncMember`(회원명), `ncBuyer`(매수인), `ncDate`(매각기일), `ncAppr`(감정가), `ncMin`(최저가), `ncBid`(낙찰가), `ncSecond`(차순위), `ncCnt`(입찰수), `ncAddr`(주소), `ncSale`/`ncJeonse`/`ncWolBo`/`ncWol`(시세, 보통 비어있음 → 조사내용에서 발췌), `ncJosa`(조사내용 전문)
- 그리드(`#ncGrid`): 사건번호, **회원등급**(회원셀의 "(등급)"), 3키 매칭 결과(낙찰 여부)

## 2) 숫자 계산 (연동값 기준 — 지어내지 말 것)
- 감정가 대비 낙찰률 = 낙찰가 ÷ 감정가 (예: 97,999,999 / 235,000,000 = 41.7%)
- 매매차익 = 매매시세 − 낙찰가
- 전세 무피/플피 = 전세가 − 낙찰가 (양수 = "무피/플피" 가능)
- 월세 수익률 = (월세 × 12) ÷ (낙찰가 − 월세보증금) × 100
- 시세는 **조사내용에서 발췌**. 범위면 보수적으로. 모든 수익·시세는 **추정치이며 세금·수리·명도비 별도** 명시.

## 3) 제목 (★회원 등급 반드시 언급, 후보 3개)
- 구성요소: **회원등급** + 핵심 숫자(감정가 대비 %, 차익/무피) + 감탄/후킹
- 예시(형식 참고, 실제 숫자로 채움):
  - "[프리미엄PT-실버] 회원님, 감정가 42%에 낙찰 — 전세만 놔도 무피!"
  - "5명 경쟁 뚫고 일산 아파트 9,800만원 낙찰 (실버등급 회원 실화)"
  - "이래서 경매 배웁니다 — [등급] 회원 낙찰 성공, 시세보다 ○○만원 싸게"
- 금지: 매수인 실명, 전화번호, "확정 수익/보장", 허위.

## 4) 본문 구조
1. **후킹 도입**(1~2줄): 부러움/호기심 유발
2. **낙찰 요약**: 주소·사건번호 / 감정가·최저가·**낙찰가(감정가 대비 %)**·입찰 N명·차순위
3. **수익 시나리오**: 매매차익 / 전세 무피·플피 / 월세 수익률 (추정·비용 별도 명시)
4. **현장 조사 요약**: 조사내용에서 **'좋은 정보'만 3~5줄로 요약 어필**(입지·수요·전월세 잘나감·개발호재 등). 부정/리스크는 과하지 않게. **부동산 전화번호는 블러/삭제**.
5. **CTA**: "부럽다면 당신도" — 과하지 않게 상담/수업 유도.

## 5) 증빙 이미지 (★상단)
- 글 **최상단**에 **옥션 사건조회(매각결과 포함) 캡처**를 배치 = "실제로 낙찰됐다"는 **증빙**.
- 이 캡처는 크롤(`06.nc.py` capture_detail)이 서버 `00.imageup/report_shots/nc_{사건}.png` 로 저장함. 매니저 카드 상단에 표시/삽입되어야 함(별도 빌드로 서빙+삽입).

## 필수 규칙 (준수)
- 매수인 이름 **성만 남기고 마스킹**(이현민 → 이○○). 연락처·주민번호 절대 노출 금지.
- 조사내용 내 **부동산 전화번호 블러/삭제**.
- 수익·시세는 **추정 + 세금·수리·명도비 별도** 명시. 확정수익 보장·허위·과장 금지.
- 숫자는 매니저 연동값(코드계산) 사용. 없는 값 지어내지 않음(모르면 생략/추정 표기).

## 출력
- **제목 후보 3개** + **본문 1안**(마크다운). 사용자 확인/선택 후 카드 제목칸·본문에 주입.
