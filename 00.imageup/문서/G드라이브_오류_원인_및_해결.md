# G:\ 오류 원인 분석 및 해결

## 1. 원래 설정 (유지)

- **저장 경로:** `G:\내 드라이브\MAPS\mapsimage`
- 웹 구글 드라이브에 이미지가 등록되는 구조와 **동일한 경로**로 두는 것이 맞습니다.
- 경로를 바꾸면, 웹과 다른 위치에 저장되므로 **원래 경로를 유지**하고, **G: 가 동작하도록** 해결해야 합니다.

---

## 2. 오류 메시지

```
FileNotFoundError: [WinError 3] 지정된 경로를 찾을 수 없습니다: 'G:\\'
```

의미: **이 PC에서 G: 드라이브를 찾을 수 없다.**

---

## 3. 원인 분석

| 가능한 원인 | 설명 |
|-------------|------|
| **1) Google Drive for Desktop 미설치** | 구글 드라이브 앱이 없으면 G: 로 잡힌 드라이브가 생기지 않음. |
| **2) 드라이브 문자가 G: 가 아님** | 앱은 있는데 "내 드라이브"를 D: 또는 다른 문자로 잡았을 수 있음. |
| **3) "내 드라이브"가 폴더로만 동기화** | 드라이브 문자 없이 `C:\Users\...\Google Drive\내 드라이브` 같은 **폴더**로만 동기화되면 G: 는 없음. |
| **4) PC/계정이 다름** | 예전에 G: 로 쓰던 PC와 지금 PC가 다르거나, 계정/설정이 달라서 이 PC에는 G: 가 없을 수 있음. |

정리하면, **"G: 드라이브가 이 PC에 존재하지 않거나 접근 불가"** 가 원인입니다.

---

## 4. 해결 방법 (원인 제거)

### A. Google Drive for Desktop 설치 및 G: 로 설정

1. [Google Drive for Desktop](https://www.google.com/drive/download/) 설치.
2. 설치 후 설정에서:
   - **"내 드라이브"** 동기화 사용.
   - **드라이브 문자를 G: 로 지정** (가능한 경우).
3. 로그인 후 **파일 탐색기**에서 **G:\** 가 보이는지 확인.
4. **G:\내 드라이브** 아래에 **MAPS → mapsimage** 폴더 구조가 있는지 확인 (없으면 만듦).

이렇게 되면 `G:\내 드라이브\MAPS\mapsimage` 가 생기고, run.py 는 **원래 경로 그대로** 동작합니다.

### B. "내 드라이브"가 폴더로만 있을 때 (G: 없음)

- 동기화 경로가 예: `C:\Users\MJ경매-이쁜언니\Google Drive\내 드라이브\...` 인 경우,
- **G: 드라이브**는 없고 **폴더만** 있는 상태입니다.
- 이 경우 두 가지 선택지:
  1. **Google Drive for Desktop 설정**에서 "내 드라이브"를 **드라이브 문자 G: 로** 사용하도록 변경 (지원되는 경우).
  2. 변경이 어렵다면, **G: 를 폴더에 연결**:
     - 관리자 CMD에서:  
       `subst G: "C:\Users\MJ경매-이쁜언니\Google Drive\내 드라이브"`  
       (실제 "내 드라이브" 경로는 PC에 맞게 확인 후 입력)
     - 그러면 `G:\MAPS\mapsimage` 로 접근 가능한지 확인 후, run.py 의 경로가 `G:\내 드라이브\MAPS\mapsimage` 인지 `G:\MAPS\mapsimage` 인지 실제 구조에 맞춰 조정.

### C. 드라이브 문자가 G: 가 아닐 때 (예: D:)

- 파일 탐색기에서 "내 드라이브"가 **D:** 등 다른 문자로 보이면,
- **옵션 1:** Google Drive for Desktop 설정에서 드라이브 문자를 **G:** 로 바꿀 수 있으면 변경.
- **옵션 2:** 바꿀 수 없으면, **subst** 로 G: 를 그 드라이브 경로에 연결:  
  `subst G: D:\`  
  (또는 "내 드라이브"의 실제 경로가 D:\ 내 드라이브 라면 `subst G: "D:\내 드라이브"` 등으로 조정)

---

## 5. 요약

- **원래 경로:** `G:\내 드라이브\MAPS\mapsimage` → **그대로 유지**하는 것이 맞고, 웹 등록과 일치합니다.
- **1차원적으로 경로만 바꾸지 말고**, **원인(G: 드라이브가 이 PC에 없음)** 을 해결해야 합니다.
- 해결: 이 PC에서 **Google Drive for Desktop** 을 쓰고, **"내 드라이브"가 G: 로 보이도록** 설치/설정하거나, **subst** 로 G: 를 해당 경로에 연결하면 됩니다.

위 설정이 되면 run.py 는 수정 없이 **원래 경로**로 동작합니다.
