# 카카오톡 연동 자동화 도입 및 커스텀 템플릿(로컬 서버 방식)

요청하신 피드백을 반영하여 설계 플랜을 업데이트했습니다. **자동/반자동 전송 설정 환경 탭**과 **공용/개별(회원별) 커스텀 템플릿 추가/편집 기능**이 포함되었습니다.

## User Review Required

> [!IMPORTANT]
> 로컬 서버 구동 환경
> 웹(MAPS)에서 로컬 PC에 띄워진 파이썬 프로그램(백그라운드 서버)으로 통신(`http://localhost:8000`)하여 매크로를 구동하는 방식입니다.
> 파이썬 매크로 모듈(`kakao_server.py`)은 제공해 드리며, 추후 각 담당자 PC에서 한 번씩 라이브러리를 설치하고 구동하시면 됩니다.

## Proposed Changes

### 1. 데이터베이스 갱신 (Google Apps Script)
#### [MODIFY] `Code.js` 및 `SheetDB.js`
- **회원 정보 확장:** `MEMBER_HEADERS` 배열 끝(AA열)에 `kakao_name` 컬럼 추가.
- **카카오 템플릿 시트 도입:** 기존 텔레그램 `msg_templates`와 분리되거나 호환되게끔, `kakao_templates` 시트 논리 추가. (속성: `템플릿명`, `내용`, `대상자구분(공용/개별)`, `대상member_id`)
- **환경 설정 항목:** `KAKAO_AUTO_ENTER` (자동 엔터 전송 여부) 값을 DB Property에서 관리하는 로직 추가.

### 2. 환경 설정: 카카오전송 탭 구성
#### [MODIFY] `index.html` & `js-app.html`
- 기존 `[환경설정 > 텔레그램]` 탭 옆에 새로 **[카카오전송]** 탭 추가.
- 해당 화면에 **"자동 전송 모드(엔터키 발송) 활성화"** ON/OFF 설정 저장 구현. 매크로 서버로 호출 시 이 상태값을 함께 전달해 파이썬에서 엔터를 칠지 말지 결정합니다.

### 3. 메시지 템플릿 시스템 고도화 (공용/개별 메시지)
#### [MODIFY] `index.html` & `js-app.html` (입찰 모달 하단 메시지 탭 구역)
- 기존 하드코딩 되어있던 '추천물건안내', '입찰가 안내' 버튼들을 **DB에서 불러온 템플릿 버튼**들로 동적 렌더링되게 변경.
- 선택된 물건의 `member_id`에 해당하는 **[개별 템플릿]**과 전체 **[공용 템플릿]**만을 합쳐서 탭 버튼으로 렌더링합니다.
- **[➕ 새 메시지 추가] 버튼 추가:** 클릭 시 팝업을 띄워 이름, 내용, "공용/개별" 여부를 선택해 추가합니다.
- **[✏️ 편집하기] 버튼 추가:** 현재 보고 있는 메시지 탭의 내용을 모달로 즉시 띄워 수정하고 바로 저장할 수 있는 기능을 구현합니다.

### 4. 카카오톡 연동 UI 및 매크로 전송
#### [MODIFY] `index.html` & `js-app.html`
- **회원관리:** 회원 정보 수정 폼에 `kakao_name` 인풋을 추가.
- **입찰 모달 메시지 탭:** 복사 버튼 주위에 **[💬 카카오톡 전송]** 버튼과 `kakao_name` 인풋 위치. (해당 정보 수정 시 즉시 회원 DB에 반영)
- 전송 클릭 시: 로컬 스크립트 실행 (`fetch("http://localhost:8000/send_kakao", { ... })`)

#### [NEW] `kakao_server.py`
- 파이썬으로 작성된 간단한 서버 겸 매크로. 
- API 요청을 받으면 클라이언트(MAPS)가 넘겨준 `auto_enter` 옵션에 따라:
  - `True`: 바탕화면 카톡 창 불러오기 -> 연락처(이름) 검색 -> 내용 붙여넣기 -> **Enter 전송**
  - `False`: 바탕화면 카톡 창 불러오기 -> 연락처(이름) 검색 -> 내용 붙여넣기까지만 수행 (대기)
