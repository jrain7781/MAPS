# MAPS 카카오톡 연동 기능 구현 매뉴얼 (MANUAL)

이 문서는 MAPS 시스템 내에 추가된 **회원 대상 카카오톡 메시지 자동화 기능**과 **동적 템플릿 로직**에 대하여 요약하여 설명합니다.

---

## 1. 시스템 구조 개요
브라우저용 자바스크립트는 보안상 로컬 데스크탑 응용프로그램을 직접 제어할 수 없습니다. 따라서 웹에서 보내는 HTTP POST 요청을 받아들여 실제로 키보드를 조작해줄 중간 다리가 필요합니다. 
* **구조**: `MAPS 화면(Web)` ➔ Fetch 요청(HTTP 8000번) ➔ `kakao_server.py (로컬 파이썬)` ➔ PyAutoGUI 단축키 매크로 ➔ `카카오톡(앱)`

## 2. 신규 기능 요약 및 설명

### [1] 카카오톡 템플릿 관리 모듈 (프론트/백엔드)
- **위치**: 구글 시트 데이터베이스 내 `kakao_templates` 시트
- **설명**: 기존처럼 JS 코드에 하드코딩 되어 있던 '추천물건안내', '입찰가안내'를 데이터베이스 연동 방식으로 완전 개편했습니다. 
- **특징**:
  - `공용(Public)` 및 `개별(Individual)` 속성을 지니며 회원 전용 메시지 템플릿을 생성할 수 있습니다.
  - 모달 상단 영역에서 실시간으로 [➕추가], [✏️편집], [삭제]를 수행할 수 있습니다.
  - 텍스트 내에 `{{사건번호}}`, `{{법원}}`, `{{이름}}`, `{{입찰가}}` 등을 치환자로 작성해 두면, 전송 전 해당 물건의 상세 정보로 자동 치환됩니다.

### [2] 회원별 [카카오_이름] 연동
- **위치**: 회원관리 메뉴 데이터베이스 (`MEMBER_HEADERS`, `ITEM_MEMBER_HEADERS`)
- **설명**: 회원가입 명부 상의 실명과 카카오톡 사용자 닉네임이 다른 경우가 매우 빈번함에 따라 이를 일치시킬 수 있는 신규 컬럼 `kakao_name`(카카오명)이 신설되었습니다.
- **특징**:
  - 회원관리 리스트 목록 및 모달 창 내에서 확인 및 수정이 가능합니다.
  - 해당 필드를 비워두게 될 시 1차적으로는 휴대폰 번호, 2차적으로는 가입자명으로 카카오톡을 찾습니다.
  - 회원 목록 테이블 우측 끝에 [💬 카톡표시 아이콘]이 나타나며 매칭 여부를 시각적으로 확인할 수 있습니다.

### [3] 카카오 전송 옵션 분리 (환경설정)
- **위치**: 상단 환경설정 톱니바퀴 > **[카카오전송] 탭**
- **설명**: 카카오톡으로 발송 시 엔터(Enter) 키까지 자동으로 쳐줄지에 대한 제어 장치입니다.
- **특징**:
  - 민감한 개인 정보(낙찰 금액, 계좌 번호 등)가 포함된 경우 발송자가 직접 한 번 더 확인하고 보내길 희망할 수 있으므로 옵션을 부여했습니다.
  - 상태값은 `ScriptProperties`(서버 속성)에 저장되어 유지됩니다.

## 3. 핵심 코드(모듈) 작동 로직 요약

- **`js-app.html::parseKakaoTemplate()`**:
  HTML 모달에서 현재 띄워진 물건 정보 객체를 받아 치환자 `{{~~}}` 와 정규표현식을 기반으로 메시지를 렌더링(치환)합니다.
  
- **`js-app.html::sendToKakao()`**:
  선택된 회원의 식별자(`kakao_name` 우선순위)와 렌더링 된 템플릿 텍스트를 JSON 바디에 묶어 로컬 서버(`http://127.0.0.1:8000/send_kakao`) 로 Fetch POST를 쏩니다. 실패 시 Try/Catch를 이용해 서버 미구동 상태를 프론트에 알림합니다.

- **`kakao_server.py::KakaoHandler`**:
  받아온 타겟의 이름과 메시지 텍스트를 `pyperclip`을 이용해 클립보드에 적재하고, `pyautogui`로 `Ctrl+F`(채팅방 검색), `Ctrl+V`(붙여넣기 및 열기), `Enter` 등을 조작하여 최종 전송합니다.
